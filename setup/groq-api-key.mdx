RepoHunt uses the Groq API to generate one-sentence AI summaries for the top search results. A valid Groq API key is required to run searches — without one, the key modal blocks access to the app.

## Why a key is needed

After every search, RepoHunt sends the top 8 ranked repositories to `https://api.groq.com/openai/v1/chat/completions` using the `llama-3.1-8b-instant` model. This call requires a Groq API key sent in the `Authorization` header:

```http
Authorization: Bearer gsk_...
```

The key is never routed through any intermediate server. Your browser sends the request directly to Groq.

## Get a free Groq key

Groq offers a free tier that is sufficient for normal RepoHunt usage.

<Steps>
  <Step title="Create a Groq account">
    Go to [console.groq.com](https://console.groq.com) and sign up. You can use a Google account or an email address.
  </Step>
  <Step title="Open API Keys">
    In the left sidebar of the Groq Console, click **API Keys**.
  </Step>
  <Step title="Create a new key">
    Click **Create API Key**, give it a descriptive name (e.g., `repohunt`), and click **Submit**.
  </Step>
  <Step title="Copy the key">
    Copy the key immediately — it starts with `gsk_`. Groq does not display the full key again after you close the dialog.

    <Warning>
      Store the key somewhere safe before closing the dialog. If you lose it, you will need to create a new one.
    </Warning>
  </Step>
</Steps>

## Enter the key in RepoHunt

On first load, if no key is found in `localStorage`, a modal appears in the center of the screen:

- **Prompt:** "enter your groq api key"
- **Input placeholder:** `gsk_...`
- **Note displayed:** "stored locally, never transmitted"

Paste your key into the input and click **Save →** or press `Enter`. The modal closes and the app becomes fully interactive.

<Note>
  The modal can be dismissed by clicking the backdrop — but only if a key is already saved. If no key is stored yet, you must enter one before you can use the app.
</Note>

## How the key is stored

RepoHunt writes the key to your browser's `localStorage` under the key name `repohunt_groq_key`:

```javascript
localStorage.setItem('repohunt_groq_key', val);
```

On every page load, the app reads it back:

```javascript
const stored = localStorage.getItem('repohunt_groq_key');
if (stored) {
  GROQ_KEY = stored;
  document.getElementById('key-overlay').classList.add('hidden');
}
```

This means you only need to enter the key once per browser. The key persists across page refreshes and browser restarts until you clear `localStorage` or explicitly change it.

<Info>
  `localStorage` is scoped to the origin. If you open `repohunt2.html` from the filesystem (`file://`) in one browser and from a local server (`http://localhost:8000`) in another, they use separate `localStorage` stores and you will need to enter the key in each context.
</Info>

## Change or update the key

To update your key at any time, click **⚙ change api key** — the small button above the filter toggle. This re-opens the key modal with the current key pre-filled in the input.

Edit or replace the value and click **Save →** (or press `Enter`) to save the new key.

## Remove the key

To remove the stored key entirely, open your browser's DevTools, go to **Application → Local Storage**, find the entry for `repohunt_groq_key`, and delete it. The next page load will show the key modal again.

## Security considerations

<Warning>
  Never commit your Groq API key to a Git repository. If you fork RepoHunt or check in any local changes, make sure `repohunt2.html` does not contain a hardcoded key.
</Warning>

<AccordionGroup>
  <Accordion title="Key format">
    All Groq API keys start with `gsk_`. If your key does not match this prefix, it was not generated by the Groq Console and will be rejected.
  </Accordion>
  <Accordion title="Where the key is sent">
    The key is sent only in the `Authorization` header of requests to `https://api.groq.com/openai/v1/chat/completions`. It is never sent to GitHub or any other service.
  </Accordion>
  <Accordion title="localStorage vs. environment variables">
    Because RepoHunt has no backend or build step, `localStorage` is the only available persistence mechanism. The key is not embedded in the HTML source at any point — it lives exclusively in your browser's storage.
  </Accordion>
  <Accordion title="Shared computers">
    On a shared or public computer, clear the key from `localStorage` after each session using **Application → Local Storage** in DevTools.
  </Accordion>
</AccordionGroup>

## What happens without a valid key

If no key is stored, the modal is shown on page load and the search button is unreachable until a key is entered. If you attempt to search while the modal is open, the modal blocks interaction with the rest of the page.

If a key is stored but is invalid or revoked, the Groq API call will fail. RepoHunt handles this gracefully: search and scoring continue to work normally, and each card displays the repository's own GitHub description in place of an AI summary.
