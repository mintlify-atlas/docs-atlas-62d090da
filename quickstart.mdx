## Prerequisites

- A modern web browser (Chrome, Firefox, Safari, or Edge)
- A free [Groq account](https://console.groq.com) for AI-powered summaries
- No Node.js, npm, build tools, or backend required

## Get a free Groq API key

RepoHunt uses Groq to generate one-sentence AI summaries for each repository. A Groq API key is required to unlock the search interface.

<Steps>
  <Step title="Create a Groq account">
    Go to [console.groq.com](https://console.groq.com) and sign up for a free account.
  </Step>
  <Step title="Navigate to API Keys">
    In the Groq console, click **API Keys** in the left sidebar.
  </Step>
  <Step title="Create a new key">
    Click **Create API key**, give it a name (e.g. `repohunt`), and copy the generated key. It starts with `gsk_`.
  </Step>
</Steps>

<Note>
  Groq's free tier is sufficient for normal RepoHunt usage. Your key is stored only in your browser's `localStorage` under `repohunt_groq_key` — it is never sent to any server other than `api.groq.com`.
</Note>

## Run RepoHunt locally

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Sujosh-Jana/RepoHunt.git
    cd RepoHunt
    ```
  </Step>
  <Step title="Open the app in your browser">
    RepoHunt is a single HTML file — no build step needed. Open `repohunt2.html` directly:

    <Tabs>
      <Tab title="macOS">
        ```bash
        open repohunt2.html
        ```
      </Tab>
      <Tab title="Linux">
        ```bash
        xdg-open repohunt2.html
        ```
      </Tab>
      <Tab title="Windows">
        ```bash
        start repohunt2.html
        ```
      </Tab>
      <Tab title="Any OS">
        Double-click `repohunt2.html` in your file manager, or drag it into your browser.
      </Tab>
    </Tabs>
  </Step>
  <Step title="Enter your Groq API key">
    On first load, a modal prompts you for your Groq API key. Paste your `gsk_...` key into the input and click **Save →** (or press Enter).

    The key is saved to `localStorage` and the modal closes. You won't need to enter it again in this browser.
  </Step>
</Steps>

## Run your first search

With the app open and your key saved, type a query in the search bar and press **Search →** or hit Enter.

**Try these example queries:**

| Query | What you'll find |
|---|---|
| `trading bot` | Algorithmic trading and backtesting tools |
| `AI agent framework` | Autonomous agent orchestration libraries |
| `Python web scraper` | HTML parsing and crawling libraries |
| `Unity save system` | Serialization and persistence tools for Unity |
| `realtime chat app` | WebSocket and messaging implementations |

You can also click any of the quick-search chips below the search bar to pre-fill a query instantly.

## Understand your results

Each result card shows:

| Element | Description |
|---|---|
| **Score badge** | A number (0–100) and letter grade (S/A/B/C/D) based on quality signals |
| **Tags** | Programming language, star count, fork count, last-updated time |
| **AI Analysis** | A one-sentence Groq-generated summary for the top 8 results |
| **Topics** | GitHub topic tags from the repository |
| **View on GitHub** | Direct link to the repository |

### Score grades

| Grade | Score | Meaning |
|---|---|---|
| **S** | ≥ 85 | Exceptional across all signals |
| **A** | ≥ 70 | High quality, actively maintained |
| **B** | ≥ 55 | Good quality with minor gaps |
| **C** | ≥ 40 | Average — worth checking manually |
| **D** | < 40 | Low signals — approach with caution |

## Try the filters

Click **Filters** to refine your results:

<AccordionGroup>
  <Accordion title="Language">
    Filter to repositories written in a specific language: Python, JavaScript, TypeScript, Go, Rust, C#, Java, C++, Ruby, or Swift.
  </Accordion>
  <Accordion title="Minimum Stars">
    Set a minimum star threshold: 50+, 100+, 500+, 1k+, or 5k+. Useful for filtering out very new or niche projects.
  </Accordion>
  <Accordion title="Last Updated">
    Limit results to repos pushed within the last 1, 3, 6, or 12 months. Helps surface actively maintained projects.
  </Accordion>
  <Accordion title="Exclude Forks">
    Switch from **Include forks** to **No forks** to see only original repositories. This adds `fork:false` to the GitHub query.
  </Accordion>
</AccordionGroup>

<Tip>
  A good starting combination for quality results: set **Min Stars** to `100+`, **Last Updated** to `Last year`, and **Exclude Forks** to `No forks`.
</Tip>
